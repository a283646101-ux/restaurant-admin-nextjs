# 多门店架构心智模型

从单店到多店扩展，必须**数据隔离 + 切换无缝**。

### 1. 数据模型设计
- 核心表加 store_id 字段（dishes、orders、users 可选绑定默认店）。
- stores 表：id, name, address, business_hours, logo。
- 用户默认门店（本地存储 + 可切换）。

### 2. 前端切换逻辑
- 首页顶部门店选择器（下拉或弹层）。
- 切换后刷新菜单 + 清空购物车（提示用户）。
- 订单自动带当前 store_id。

### 3. 后台管理
- 超级管理员可管理所有门店，店长只能看本店数据（RLS 严格限制）。
- 菜品支持“全局 + 门店专属”两种模式。

### 4. 一致性与扩展
- 库存按门店独立扣减。
- 报表支持按门店筛选。

目标：未来开新店只需加一条 store 记录，无需改代码。
